# 06. 기여도 산정 공식

기여도는 **회의 활동(발언·참석)** 과 **맡은 일(액션)** 을 분리한 **2트랙 + 최종 종합** 구조로 산정한다. 기존 3축(액션 0.5 / 발언 0.3 / 참석 0.2) 비중은 그대로 보존한다.

**대상**: `meetings.meeting_type = 'regular'` (정규 회의).

---

## 점수 4종

| # | 이름 | 범위 | 공식 | 저장 |
| --- | --- | --- | --- | --- |
| ① | 회의 기여도 | user × meeting | 발언 × 0.6 + 참석 × 0.4 | `contribution_scores.meeting_score` **저장** |
| ② | 회의 종합 기여도 | user × team | Σ(① × total_minutes) / Σ(total_minutes) | 동적 |
| ③ | 테스크 기여도 | user × team | (완료율 + 마감준수) / 2 | 동적 (`action_items`에서) |
| ④ | 종합 기여도 | user × team | min(1.0, (③ × w + ② × (1−w)) × 팀장배율) | 동적 |

```
w = team_settings.final_task_weight (테스크 비중, 기본 0.5, 팀 설정에서 조정 가능)
팀장배율 = team_settings.leader_bonus_multiplier (기본 1.0, 팀원은 1.0, 1.0 캡)
```

> ①만 회의 종료 시 고정값이라 저장. ②③④는 시간 경과·액션 완료로 변하므로 조회 시점에 ①과 `action_items`에서 계산.
> 가중치 유도: 최종의 액션 0.5(=③) / 회의 0.5 내부 발언:참석 = 0.6:0.4 → 기존 0.5/0.3/0.2 보존.

---

## 트랙 1 — 회의 기여도 (①, 회의별)

```
meeting_score = 발언 축 × 0.6 + 참석 축 × 0.4
```

### 발언 축

```
발언 축 = (발언 비중 점수 + speech_consistency) / 2

speech_ratio(raw) = Σ(본인 char_count) / Σ(전체 char_count)      ← 기여도 바 표시용(%)
발언 비중 점수    = min(1.0, speech_ratio × N)                    ← 산정용 (1/N 만점)
   N = 회의 참석 참여자 수 (활성 윈도우 > 0인 전원, 발언 0인 참석자 포함)
   char_count는 team_settings.max_utterance_chars(기본 500)에서 절단 후 합산

speech_consistency = 본인 발언 agenda 수 / 진행된 agenda 수
```

→ `utterances` (user_id, char_count, agenda_id), `agendas`

- N명이 균등 발언(각 1/N)하면 만점이 되도록 정규화 — 독점하지 않아도 공정 분량이면 만점, 초과분은 1.0 캡.
- 무발언 참석자도 N에 포함 — 본인 점수는 0, 분모 N을 키워 말한 사람 만점선을 낮춤(공정).

### 참석 축

```
참석 축 = (attendance_ratio + punctuality_score) / 2

attendance_ratio  = (참여 − 자발 자리비움 − 비자발 끊김) / (실제 진행시간 − 비자발 끊김)
   실제 진행시간 = ended_at − t0_timestamp
   자발 자리비움: team_settings.presence_grace_seconds(기본 30s) 이상 이탈만 차감
punctuality_score = max(0, 1 − 지각분 / (total_minutes × punctuality_grace_ratio[기본 0.10]))
```

→ `presence_events` (disconnect_classification), `meetings` (total_minutes, t0_timestamp, ended_at), `team_settings`

- 비자발 끊김(네트워크 사고 등 본인 책임 아닌 시간)은 출석 분모·분자에서 제외.

### 발언 인식 신뢰도

```
utterances.confidence = Web Speech alternative.confidence (0~1) 그대로 저장 (null 허용)
STT 실패(anomaly_events.event_type='stt_failure') 구간 → 발언 분모·분자서 제외
   판정: onerror(network/no-speech/aborted) + onend 재시작 실패
confidence_level: 오디오손실<5% & STT실패<10% = high, ≤20% = medium, 그 외 low (운영 중 조정)
```

## 트랙 2 — 테스크 기여도 (③, 사람×팀)

회의 점수와 분리된 별도 트랙. 별도 스냅샷 테이블 없이 `action_items`에서 **조회 시점 라이브 계산**. 팀 범위는 `action_items.team_id`로 직접 집계.

```
action_score = (action_completion_ratio + deadline_adherence) / 2

완료율   action_completion_ratio = Σ(done × difficulty) / Σ(difficulty)
마감준수 deadline_adherence       = Σ(마감점수 × difficulty) / Σ(difficulty)
   대상: 본인(assignee_id) × 팀(team_id)의 확정 액션. 난이도 difficulty 상/중/하 = 3/2/1
```

→ `action_items` (team_id, assignee_id, status, completed_at, due_date, difficulty, confirmed)

### 마감점수 (일자별 공식, `team_settings.deadline_penalty_curve` 3종)

```
초과일수 = ceil((completed_at − due_at) / 1일), 정시·조기는 0

standard(기본): max(0, 1 − 0.2 × 초과일수)   1일당 0.2↓ → 정시1.0, 1일0.8, …, 5일~0.0
lenient (유함): max(0, 1 − 0.1 × 초과일수)   2일당 0.2↓ → 정시1.0, 2일0.8, …, 10일~0.0
strict        : 초과일수=0 → 1.0,  그 외(1일이라도 초과) → 0.0
```

### status별 처리

| status | 처리 |
| --- | --- |
| done | 완료 시각으로 위 공식 적용 |
| todo·in_progress + 기한 지남 | 미완료 = 0 (분모 포함) |
| todo·in_progress + 기한 전 | **평가 보류 → 완료율·마감준수 분모서 제외** |
| cancelled | 무단 취소 = 0 (분모 포함) |
| delete | 정상 삭제 = 분모 제외 |

## 측정 불가 항목 정규화

```
축 점수   = 측정 가능한 지표들의 평균
meeting_score = Σ(측정가능 축 × 가중치) / Σ(측정가능 축 가중치)
```

측정 불가 지표/축(액션 0개·무발언 등)은 0점이 아니라 분모에서 제외. 제외 항목은 `contribution_scores.excluded_indicators`에 기록.

| 지표 | 측정 가능 조건 |
| --- | --- |
| 완료율 / 마감준수 | Σ difficulty > 0 (평가 대상 액션 존재) |
| 발언 비중 | 활성 윈도우 > 0 AND 전체 글자수 > 0 |
| 발언 안정성 | 진행된 agenda 수 > 0 |
| 출석 | (실제 진행시간 − 비자발 끊김) > 0 |
| 정시 도착 | 첫 join 기록 존재 |

---

## 회의 종합 기여도 (②, 누적)

```
② = Σ(meeting_score × total_minutes) / Σ(total_minutes)
```

긴 회의일수록 비중이 커야 공정하므로 회의 시간으로 가중 평균(당근 온도 방식).

**누적 대상 회의**

```
meeting_type = 'regular' 만 (partial 제외)
실제 진행시간 ≥ team_settings.min_meeting_minutes (기본 5분, 미만 제외)
사유 결석(exclude) 회의 제외
meeting_score = null(측정 불가) 회의 → 분모·분자 모두 제외
```

---

## 결석 처리

```
무단 결석 (presence_events 0건, 한 번도 입장 X)
   → ① = 0, 누적(②)에 그대로 포함

사유 결석 (meeting_absences 등록 + 팀장 승인)
   → team_settings.absent_meeting_handling:
        exclude (기본) : 그 회의 ① 미산정, 누적 분모·분자서 제외 (불가항력 보호)
        zero           : ① = 0, 누적 포함
        attendance_only: 참석 축만 0, 발언 축은 정상 산정
```

> 사유 결석(exclude)은 "0점"이 아니라 "없던 일" — 참석한 회의들로만 ② 산정.
> 테스크(③)는 회의 출석과 무관 — 결석해도 맡은 액션 완료분은 정상 인정.

## 팀 설정값 → 식 연동

| `team_settings` | 기본 | 연동 식 |
| --- | --- | --- |
| punctuality_grace_ratio | 0.10 | 정시: 허용 지각 = total_minutes × 비율 |
| presence_grace_seconds | 30 | 출석: 자리비움 차감 기준 |
| max_utterance_chars | 500 | 발언: char_count 절단 |
| deadline_penalty_curve | standard | 마감점수 곡선 선택 (standard/lenient/strict) |
| absent_meeting_handling | exclude | 사유 결석 처리 |
| min_meeting_minutes | 5 | 누적 반영 대상 필터 |
| final_task_weight | 0.50 | ④ 최종 합산 테스크 비중 w |
| weight_speech_in_meeting | 0.60 | ① 회의 내 발언 가중치 (출석과 합 1.0) |
| weight_attend_in_meeting | 0.40 | ① 회의 내 출석 가중치 |
| leader_bonus_multiplier | 1.00 | ④ 팀장 배율 |
| contribution_visibility | team | 점수 UI 공개 범위 (식 아님) |

> 발언:참석 가중치(`weight_speech_in_meeting`/`weight_attend_in_meeting`, 기본 0.6/0.4)와
> 테스크 비중(`final_task_weight`)은 **팀 설정에서 조정 가능**(팀장 권한). 난이도 가중치만 전역 고정.
