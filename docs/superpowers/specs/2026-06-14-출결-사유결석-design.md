# 출결 표시 + 사유 결석 동의 — 설계

작성일: 2026-06-14

## 목적

종료된 회의의 **출결(출석/지각/결석)을 시각적으로 확인**할 수 있게 하고, 결석에 대해 **사유를 입력해 팀원 과반이 동의하면 출석으로 인정**한다. 단, 인정은 **출결 표시(시각)에만** 반영하고 **기여도 점수는 건드리지 않는다**.

## 배경 (기존 구조)

- 회의 종료 시 참여자별 `ContributionScore`가 저장된다: `attendance_ratio`, `punctuality_score`(지각 시 감점), `meeting_score` 등.
- **기여도 산정은 외부 엔진([cc-team-8/Contribution](https://github.com/cc-team-8/Contribution))이 전담**한다. 우리 서버는 산정 로직을 수정하지 않는다.
- 출결 표시는 추가 저장 없이 입장 기록(`PresenceEvent`)과 저장값에서 파생한다.

## 설계 결정 (확정)

1. **점수는 건드리지 않는다.** 사유 결석이 인정돼도 기여도 점수에서 빼주지 않는다 — 실제 참여(발언·참석)가 없었으니 점수가 낮은 건 정당하며, 외부 엔진이 그대로 산정한다. "출석 인정"은 **출결 탭의 표시(🟢 출석 인정)에만** 반영한다.
2. **동의 정족수** = 그 결석 건의 본인만 제외한 나머지 현재 활성 팀원의 절반 이상. `필요 동의수 = ceil((현재_활성_팀원수 − 1) / 2)`. 결석자가 여러 명이어도 각 결석 건은 독립적으로, "그 건의 본인만" 모수에서 제외한다(다른 결석자는 동의 가능).
3. **출결 판정** = 입장 기록(`PresenceEvent`) 기준 — 산정 경로(외부/로컬)와 무관하게 견고. 지각 여부만 `ContributionScore.punctuality_score` 보조.
4. **사유 인정 대상** = 결석만. 지각/출석은 표시 전용.
5. **UI** = 회의 상세에 '출결' 탭 신설.

## 데이터 모델 (신규 2개)

```
MeetingAbsence  — 결석 사유 1건 (회의 × 사용자)
  id            PK
  meeting_id    FK
  user_id       FK (결석자 본인)
  reason        text (사유)
  status        enum: pending | approved | rejected
  created_at
  UNIQUE(meeting_id, user_id)   -- 회의당 본인 사유 1건

AbsenceConsent  — 동의 1건 (중복 동의 방지)
  id            PK
  absence_id    FK
  voter_id      FK (동의한 팀원)
  created_at
  UNIQUE(absence_id, voter_id)
```

출결 상태(출석/지각/결석)는 별도 테이블에 저장하지 않고 입장 기록·저장값에서 파생한다.

## 출결 판정 (파생 규칙)

| 조건 | 표시 |
|------|------|
| 입장(join/reconnect) 기록 없음 | 결석 |
| 입장 O, `punctuality_score < 1` (팀 설정 유예 초과) | 지각 |
| 그 외 | 출석 |
| 위가 결석이고 해당 회의에 `status=approved`인 `MeetingAbsence` 존재 | 출석 인정 |

- **결석 판정은 `PresenceEvent`(입장 기록) 기준** — 외부 엔진 경로는 결석을 `attendance_ratio=0`, 로컬 경로는 `null`로 저장해 서로 다르므로, 입장 기록 유무로 판정해야 견고하다.
- 지각 시간(분)은 첫 입장 오프셋에서 도출해 표기("지각 12분").
- 지각 여부는 `punctuality_score`(로컬 경로) 기준 — 외부 엔진 경로는 미제공(null)이라 지각이 출석으로 보일 수 있음(알려진 한계).

## 플로우

1. **사유 입력** — 결석으로 판정된 **본인**이 사유 입력 → `MeetingAbsence(status=pending)` 생성.
2. **동의** — 결석자를 제외한 **활성 팀원**이 동의 → `AbsenceConsent` 1건. 멱등(이미 동의했으면 무시).
3. **자동 승인** — 동의 수가 `ceil((활성_팀원수 − 1)/2)`에 도달하면 `status=approved`로 전환. **점수는 건드리지 않고 출결 표시만 '출석 인정'으로** 바뀐다.
4. **시한** — 입력·동의는 회의가 **종료(ended)·유효(미무효)** 상태일 때 가능. (meeting에 별도 확정 플래그가 없어 confirm 시한은 후속 과제로 보류.)

### 상태 전이

```
(없음) --본인 사유 입력--> pending --동의 정족수 도달--> approved (출결 표시만 '출석 인정')
```

## API (신규)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/meetings/:id/attendance` | 출결 목록(파생) + 각자 사유/동의 현황 |
| POST | `/api/meetings/:id/absences` | 본인 결석 사유 입력 (종료·유효 회의, 본인이 결석일 때만) |
| POST | `/api/absences/:id/consent` | 동의 (결석자 제외 팀원, 멱등, 정족수 도달 시 자동 승인) |

`GET /attendance` 응답: `meeting_id`, `consent_required`, `members[]{ user_id, name, profile_image_url, status(present|late|absent|excused), late_minutes, absence?{ id, reason, status, consent_count, my_consent } }`

### 권한·검증
- 사유 입력: 요청자 == 결석자 본인, 회의 status == ended && 미무효, 본인이 실제 결석(입장 기록 없음)일 때만.
- 동의: 요청자가 그 팀의 활성 멤버 && 결석자 본인이 아님. 본인 결석 건에 자기 동의 불가.
- **기여도 산정 로직은 수정하지 않는다** — 외부 엔진(cc-team-8/Contribution)이 전담.

## UI (출결 탭)

회의 상세 탭에 `attendance` 추가: `아젠다 · 발언기록 · 출결 · 결정사항 · 회의요약`.

```
출결 현황
 ● 김철수     🟢 출석
 ● 이영희     🟡 지각 12분
 ● 박민수     🔴 결석        [사유 입력]   ← 본인일 때
 ● 정지원     🟢 출석

 — 다른 멤버 결석 건 —
 최서준  🔴 결석 · 사유: 가족행사
            동의 2/3   [동의하기]
```

- 배지 색상은 기존 토큰 재사용: 출석/출석 인정 `--green`, 지각 앰버, 결석 `--coral`. 출석 인정은 "출석 인정" 라벨.
- 본인이 결석이면 사유 입력 버튼(모달), 타인 결석 건엔 동의 버튼 + `N/M` 현황. 이미 동의했으면 비활성.
- 입력된 사유는 하단에 멤버별로 모아 표시하고, 인정된 건은 "인정됨" 표기.

## 범위 밖 (YAGNI)

- **기여도 점수 조정 없음** — 사유 결석도 점수는 외부 엔진이 그대로 산정(낮음 유지). 출결 표시에만 반영.
- 명시적 반대표 없음 — 동의만 카운트, 정족수 미달이면 `pending` 유지.
- 지각 사유 인정 없음 — 결석만.
- 회의 확정(confirm) 시한 없음 — 종료·유효 회의면 가능. confirm 시한은 후속.
- 알림(동의 요청/승인) 연동은 후속 과제.
- `rejected`는 스키마에만 두고 이번 구현에선 자동 전이 없음(향후 명시적 거부용 예약).

## 테스트 관점

- 출결 파생: 입장 기록 없음→결석, punctuality<1→지각, 정시→출석, approved 사유→출석 인정.
- 정족수: 5인 팀(결석 1) → 2명 동의 시 승인, 1명이면 미승인.
- 멱등: 동일인 중복 동의 시 카운트 1.
- 승인돼도 기여도 점수는 변하지 않음(출결 표시만 변경).
- 권한: 타인 사유 입력 거부, 본인 자기 동의 거부, 미종료 회의 사유 입력 거부.
